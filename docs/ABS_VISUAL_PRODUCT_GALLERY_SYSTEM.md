# Asset Business Solutions (ABS)

## Visual Product Gallery System

Purpose:
Define how products are visually presented across the ABS website.

The website acts as a DIGITAL SHOWROOM.

Every product must be:

1. Seen clearly
2. Understood quickly
3. Compared easily
4. Configured intuitively

This system ensures consistent product storytelling across:

- Arcplus software
- Scanners
- Tags
- Services

---

# 1. Product Showcase Philosophy

ABS product pages follow the storytelling flow:

Hero → Context → Capability → Configuration → Action

This mirrors how customers evaluate technical solutions.

Users should feel like they are walking through a **digital showroom**.

---

# 2. Gallery Content Layers

Each product must have five visual layers.

Layer 1 — Hero Image  
Layer 2 — Context Image  
Layer 3 — Detail Image  
Layer 4 — Workflow Image  
Layer 5 — Configuration Visual

---

# 3. Hero Image (Primary Product Display)

Purpose:
Immediate product recognition.

Rules:

- Clean background
- Centered product
- High resolution
- Minimal text overlay

Example:

Scanner hero:
Handheld scanner isolated on light grey background.

Tag hero:
RFID tag centered on white.

Arcplus hero:
Dashboard overview.

---

# 4. Context Images (Real World Use)

Purpose:
Show the product working in real environments.

Examples:

Scanners:

- Warehouse scanning
- Field asset verification
- Inventory capture

Tags:

- Tags attached to equipment
- Outdoor infrastructure tagging

Arcplus:

- Dashboard used by asset manager
- Tablet view during field inspection

---

# 5. Detail Images

Purpose:
Build technical credibility.

Examples:

Scanners:

- Scan trigger
- Screen interface
- Charging dock

Tags:

- Material surface
- Adhesive backing
- Mounting mechanism

---

# 6. Workflow Images

Purpose:
Explain operational value.

Example sequence:

Tag → Scan → Upload → Manage → Report

These can be:

- diagrams
- staged photos
- UI overlays

---

# 7. Configuration Visuals

Used in RFQ product pages.

Examples:

Scanner configuration:

- connectivity options
- scanning modes

Tag configuration:

- material
- mounting
- read range

Visual aids help users understand configuration choices.

---

# 8. Product Gallery Layout

Standard gallery layout:

LEFT:
Large product image viewer

RIGHT:
Product summary

Below:
Scrollable visual story sections

Example order:

Hero image  
Context image  
Feature highlights  
Workflow diagram  
Specifications

---

# 9. Product Tile System (Homepage & Category Pages)

Products appear as tiles.

Tile Structure:

Image (dominant)  
Product name  
1-line value statement  
Explore button

Example:

Handheld RFID Scanner  
"Rapid asset capture with industrial reliability."

---

# 10. Horizontal Product Galleries

Used on homepage.

Scrolling row of large product cards.

Cards reveal more details on hover.

This encourages exploration.

---

Dashboard gallery

---

# 13. Image Production Guidelines

Lighting:
Bright, neutral.

Backgrounds:
White or light grey.

Resolution:
Minimum 2000px width.

Formats:

Primary: WebP  
Fallback: PNG

---

# 14. Consistency Rules

All product images must:

Use similar lighting  
Maintain neutral backgrounds  
Avoid heavy filters  
Focus on product clarity

---

# 15. Mobile Gallery Behaviour

Mobile gallery becomes swipe carousel.

Users swipe between:

Hero  
Context  
Details  
Workflow

---

# 16. Comparison Gallery (Optional)

Users can compare products visually.

Example:

Scanner comparison cards:

Image  
Scan range  
Connectivity  
Durability

---

# 17. Launch Minimum Requirement

To launch the site you need:

Arcplus dashboard screenshots  
1 scanner hero image  
1 scanner context image  
1 RFID tag hero image  
1 services field image

Additional galleries can expand later.
