# Asset Business Solutions (ABS)

## Website Information Architecture Map

Purpose:
Define the full structure of the ABS website including:

- navigation hierarchy
- page relationships
- user pathways
- product discovery flows

The website is designed as a **self-guided asset management showroom**.

Users must be able to move from:
Discovery → Evaluation → Action.

# 12. Conversion Pathways

Users follow one of four journeys:

Path 1 — SaaS adoption
Home → Arcplus → Pricing → Free Trial

Path 2 — Hardware procurement
Home → Scanners/Tags → Configure → RFQ

Path 3 — Service engagement
Home → Services → Service page → Request consultation

Path 4 — Training
Home → Training → Register
